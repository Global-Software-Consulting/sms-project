Snapshot of what was checked, what was found, and what was fixed before the
next production push. Companion to `NAVIGATION_BUG_RESOLUTION.md`. Future
engineers: skim this before deploying so you know which sleeping risks were
acknowledged but not paid down.

## What was found

### Fixed in this PR

1. **Hardcoded backdoor credential in `src/config/next-auth.config.ts`.**
   The `CredentialsProvider.authorize` callback accepted the literal pair
   `demo@example.com` / `Demo@123` and returned a fake authenticated user.
   The whole file is dormant scaffolding ‚Äî real auth flows through
   `apiClient` + `tokenStorage` against the sms-api backend, and there is
   no `app/api/auth/[...nextauth]/route.ts` mounting `authOptions` ‚Äî
   so today the backdoor cannot be reached. But the moment anyone wires
   NextAuth in the future, the backdoor activates silently.
   **Fix:** `authorize` is now a hard no-op that always returns `null`.
   A long comment explains the file is dormant and warns the next
   engineer not to re-introduce literal-string credential matching.

2. **Image Auto Editor had no file-size or file-count bounds.** Admin
   could upload N √ó any-size images; the browser canvas would OOM the
   tab on a 50 √ó 30 MB batch.
   **Fix:** `MAX_FILE_BYTES` (10 MB) and `MAX_BATCH_FILES` (20)
   exported from `src/lib/blog-image-editor.ts`. The admin upload
   handler in `src/app/(admin)/admin/blogs/page.tsx` now drops
   oversize files and caps the batch with a toast explaining what
   was skipped. Blast radius was already "one admin crashes their own
   tab" so this is hardening, not breach repair.

### Known, not fixed in this PR ‚Äî track these

3. **Duplicate `PayoutStatus` export across `referralsApi.ts` and
   `adminReferralsApi.ts`.** The two define different unions
   (the user-facing one has `FAILED`; the admin one doesn't). Both are
   re-exported through `src/lib/api/index.ts` causing TS2308. Today no
   consumer imports `PayoutStatus` via the barrel ‚Äî only direct imports ‚Äî
   so no runtime crash. Risk is that a future barrel import silently
   loses the `FAILED` branch and misses a real status. **Fix when
   touched:** pick one canonical type or alias one in the barrel.

4. **24 `as any` casts.** Most are admin table-column accessors
   (`(item as any)[column.key]`) ‚Äî low risk because if the column key
   is wrong it just renders `-`. Two on `home.client.tsx:139` and
   `pricing.client.tsx:156` cover unknown API response shapes; if the
   backend response format changes those will silently render blank
   instead of throwing. **Fix when touched:** add proper response types
   in the corresponding API client.

5. **`dangerouslySetInnerHTML` on raw `pageToView.content` in
   `src/app/(admin)/admin/legal/page.tsx:703`.** Admin self-XSS only ‚Äî
   an admin can stick a `<script>` in legal content and execute it as
   themselves. Lower priority since admin is the trusted party, but a
   clean fix is sanitizing with DOMPurify before render. The public
   landing legal page already escapes via the `esc` helper in
   `src/components/landing/legal-page.tsx`, so the public side is safe.

6. **No CI smoke test for the navigation bug fix.** The 7-layer stack
   documented in `NAVIGATION_BUG_RESOLUTION.md` is fragile precisely
   because nothing in CI asserts it still works. **Fix:** one
   Playwright test that clicks landing nav links and asserts that
   `document.title` advances together with the URL. Catches every
   future regression of every layer.

### Non-issues ‚Äî flagged, then dismissed

- `Custom Cache-Control headers detected` warning during `next build` ‚Äî
  for `_next/static/:path*` and `_next/image/:path*`. These are
  framework-owned routes; the override is intentional. Benign today.
- Several `.catch(() => {})` silent swallows in `contact.client.tsx`,
  `api.client.tsx`, `addons-loader.tsx`, `maintenance/page.tsx`,
  `activation/page.tsx`. Each was reviewed and is a best-effort lookup
  (site info, addons, rate limits) where falling back to a default is
  the correct behavior.
- Admin sidebar has no Grafana shortcut. Confirmed missing, but the
  user explicitly deferred this earlier ‚Äî it stays on the backlog.

  - UptimeRobot ‚Üí "View live uptime details" link on `/status`

  returns them; default fallback covers 6 of 7 (Binance intentionally

  infinite-reload.

  `javascript:` or `data:` links into the status page button.
